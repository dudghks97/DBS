JDBC 드라이버의 새로운 기능

다음 섹션에서는 Microsoft SQL Server JDBC 드라이버의 새로운 기능에 대해 설명합니다.

JDBC 드라이버 버전 3.0의 새로운 기능

Microsoft SQL Server JDBC 드라이버 3.0에는 새롭고 향상된 기능이 다양하게 들어 있습니다.

드라이버 이름

새 버전의 JDBC 드라이버는 Microsoft SQL Server JDBC 드라이버 3.0입니다.

SQL Server 2008에서 도입된 SQL Server 2008의 시간 및 날짜 데이터 형식에 대한 지원

Microsoft SQL Server JDBC 드라이버 3.0에서는 이제 다음과 같은 SQL Server 데이터 형식을 지원합니다.

  • time
  • date
  • datetime2
  • datetimeoffset

이러한 SQL Server 형식에 대한 자세한 내용은 날짜 및 시간 데이터 형식 및 함수(Transact-SQL)를 참조하십시오.

DateTimeOffset 클래스 형식과 함께 datetimeoffset 열의 값에 액세스하는 데 사용할 수 있는 함수가 추가되었습니다.

  • SQLServerCallableStatement.getDateTimeOffset
  • SQLServerCallableStatement.setDateTimeOffset
  • SQLServerPreparedStatement.setDateTimeOffset
  • SQLServerResultSet.getDateTimeOffset
  • SQLServerResultSet.updateDateTimeOffset

java.sql.Time 값을 서버에 보내는 방식 구성

sendTimeAsDatetime 연결 속성이 추가되었습니다. sendTimeAsDatetime은 java.sql.Time 값을 서버에 SQL Server time 값 또는 datetime 값으로 보내는 방식을 결정합니다.

sendTimeAsDatetime 연결 속성을 사용하는 방법은 연결 속성 설정을 참조하십시오.

java.sql.Time 형식을 서버에 보내는 방법은 java.sql.Time 값을 서버에 보내는 방식 구성을 참조하십시오.

SQLServerDataSource.setSendTimeAsDatetime을 사용하여 sendTimeAsDatetime 연결 속성의 값을 프로그래밍 방식으로 수정할 수 있습니다.

MERGE 작업에 대한 완전한 지원

다음 메서드는 이제 MERGE 작업에 대해 업데이트되는 행의 정확한 수를 반환합니다.

  • SQLServerStatement.executeUpdate
  • SQLServerStatement.getUpdateCount
  • SQLServerPreparedStatement.executeUpdate
  • SQLServerPreparedStatement.getUpdateCount(SQLServerStatement에서 상속)
  • SQLServerCallableStatement.executeUpdate(SQLServerPreparedStatement에서 상속)
  • SQLServerPreparedStatement.getUpdateCount(SQLServerStatement에서 상속)

날짜 및 시간 형식에 getBytes, setBytes 및 updateBytes가 지원되지 않음

Microsoft SQL Server JDBC 드라이버 2.0에서는 SQLServerCallableStatement.getBytes, SQLServerCallableStatement.setBytes, SQLServerResultSet.getBytes 또는 SQLServerResultSet.updateBytes를 사용하여 바이트 배열과 SQL Server 데이터 형식 date, time, datetime2 또는 datetimeoffset 간에 값을 변환할 수 있었습니다. SQL Server JDBC 드라이버 3.0에서는 이러한 메서드와 이러한 데이터 형식을 함께 사용하면 변환이 지원되지 않음을 나타내는 예외가 발생합니다.

getObject가 날짜 또는 시간 개체를 반환함

Microsoft SQL Server JDBC 드라이버 2.0에서는 SQLServerCallableStatement.getObject 또는 SQLServerResultSet.getObject를 사용하여 date, time, datetime2 또는 datetimeoffset 형식의 값을 검색할 때 java.lang.String 형식의 개체가 반환되었습니다.

SQL Server JDBC 드라이버 3.0에서는 다음과 같은 결과가 나타납니다.

  • date 형식의 값은 java.sql.Date 개체로 반환됩니다.
  • time 형식의 값은 java.sql.Time 개체로 반환됩니다.
  • datetime2 형식의 값은 java.sql.Timestamp 개체로 반환됩니다.
  • datetimeoffset 형식의 값은 microsoft.sql.DateTimeOffset 개체로 반환됩니다.

열 동작 변경 사항

SQL Server JDBC 드라이버 3.0의 일부 열에 대해 SQLServerDatabaseMetaData.getColumns는 이전 버전의 드라이버와 다른 값을 반환합니다. 자세한 내용은 SQLServerDatabaseMetaData.getColumns를 참조하십시오.

SQLServerResultSetMetaData의 다음 메서드도 동작이 변경되었습니다.

  • SQLServerResultSetMetaData.getColumnDisplaySize
  • SQLServerResultSetMetaData.getColumnType
  • SQLServerResultSetMetaData.getColumnTypeName
  • SQLServerResultSetMetaData.getScale

인터페이스

다음 인터페이스가 추가되었습니다.

  • ISQLServerCallableStatement
  • ISQLServerConnection
  • ISQLServerDataSource
  • ISQLServerPreparedStatement
  • ISQLServerResultSet
  • ISQLServerStatement

자세한 내용은 래퍼 및 인터페이스를 참조하십시오.

클래스 생성자 사용 중단 예정

SQL Server JDBC 드라이버 3.0 버전을 발표하면서 Microsoft는 일부 클래스의 생성자를 이후 버전에서 제외하기로 결정했습니다. Microsoft에서는 개발자가 이러한 형식의 인스턴스를 직접 인스턴스화하지 않기를 원합니다.

이러한 클래스의 생성자는 SQL Server JDBC 드라이버 3.0 버전에서 아직 사용됩니다.

SQL Server JDBC 드라이버 3.0 버전에서 다음 클래스에는 해당하는 인터페이스가 있습니다. 드라이버 관련 메서드나 상수에 액세스해야 하는 경우에는 클래스 대신 인터페이스를 사용합니다.

  • SQLServerCallableStatement 클래스(해당하는 인터페이스는 ISQLServerCallableStatement 인터페이스임)
  • SQLServerConnection 클래스(해당하는 인터페이스는 ISQLServerConnection 인터페이스임)
  • SQLServerPreparedStatement 클래스(해당하는 인터페이스는 ISQLServerPreparedStatement 인터페이스임)
  • SQLServerResultSet 클래스(해당하는 인터페이스는 ISQLServerResultSet 인터페이스임)
  • SQLServerStatement 클래스(해당하는 인터페이스는 ISQLServerStatement 인터페이스임)

다음 클래스는 SQL Server 관련 함수를 노출하지 않으므로 이러한 클래스의 생성자가 이후 버전에서 제외될 예정이며 해당하는 인터페이스가 없습니다.

  • SQLServerDatabaseMetaData
  • SQLServerException
  • SQLServerParameterMetaData
  • SQLServerPooledConnection
  • SQLServerResultSetMetaData
  • SQLServerSavepoint
  • SQLServerXAConnection

SQLServerDataSource.getReference에 더 이상 Password 또는 trustStorePassword가 포함되지 않음

JDBC 드라이버 3.0 이전에는 SQLServerDataSource 개체에 대해 SQLServerDataSource.setPassword가 호출된 경우 SQLServerDataSource.getReference에서 반환되는 개체에 암호가 제공되었으므로 이 개체를 사용하여 추가 연결을 만들 수 있었습니다. JDBC 드라이버 3.0에서는 SQLServerDataSource.getReference에서 반환되는 개체에 암호를 설정해야 이 개체를 사용하여 연결을 만들 수 있습니다.

또한 데이터 원본 속성을 바인딩하기 전에 SQLServerDataSource.setTrustStorePassword를 설정할 경우 연결을 가져오기 전에 SQLServerDataSource.setTrustStorePassword를 호출해야 합니다. 자세한 내용은 SQLServerDataSource.getReference를 참조하십시오.

스파스 열 지원

SQL Server JDBC 드라이버 3.0에서는 SQL Server 2008 또는 그 이상의 서버에 연결할 때 스파스 열을 지원합니다. 자세한 내용은 스파스 열을 참조하십시오.

큰 UDT(사용자 정의 형식)

8KB를 초과하는 크기의 UDT에 대한 지원이 추가되었습니다. 자세한 내용은 사용자 정의 형식을 참조하십시오.
