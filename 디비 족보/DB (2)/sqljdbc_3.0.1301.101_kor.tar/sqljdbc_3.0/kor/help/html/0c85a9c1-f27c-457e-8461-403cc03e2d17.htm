setEncrypt 메서드(SQLServerDataSource)

encrypt 속성이 사용되는지 여부를 나타내는 boolean 값을 설정합니다.

구문

public void setEncypt(boolean encrypt)

매개 변수

encrypt

클라이언트와 SQL Server 간에 SSL(Secure Sockets Layer) 암호화가 사용되면 true이고, 그렇지 않으면 false입니다.

설명

encrypt 속성이 true로 설정되어 있으면 Microsoft SQL Server JDBC 드라이버에서는 서버에 인증서가 설치되어 있는 경우 SQL Server에서 클라이언트와 서버 간에 전송되는 모든 데이터에 대해 SSL 암호화를 사용하도록 지정합니다. 기본값은 false입니다.

JDBC 드라이버에서는 SSL 핸드셰이크를 설정하려고 할 때 해당 드라이버가 실행 중인 JVM(Java Virtual Machine)을 검색합니다.

encrypt 속성이 true로 설정되어 있는 경우 Microsoft SQL Server JDBC 드라이버에서는 JVM의 기본 JSSE 보안 공급자를 사용하여 SQL Server와 SSL 암호화를 협상합니다. 기본 보안 공급자는 SSL 암호화를 성공적으로 협상하는 데 필요한 기능을 모두 지원하지 않을 수 있습니다. 예를 들어 기본 보안 공급자가 SQL Server 인증서에 사용되는 RSA 공개 키의 크기를 지원하지 않을 수 있습니다. 이 경우 기본 보안 공급자에서 오류가 발생하여 JDBC 드라이버가 연결을 종료합니다. 이 문제를 해결하려면 다음 중 하나를 수행하십시오.

  • 보다 작은 RSA 공개 키를 사용하는 서버 인증서를 사용하여 SQL Server를 구성합니다.
  • "<java-home>/lib/security/java.security" 보안 속성 파일에서 다른 JSSE 보안 공급자를 사용하도록 JVM을 구성합니다.
  • 다른 JVM을 사용합니다.

encrypt 속성이 지정되어 있지 않거나 false로 설정되어 있으면 드라이버에서는 SQL Server에서 SSL 암호화를 지원하도록 지정하지 않습니다. SQL Server 인스턴스가 SSL 암호화를 사용하도록 구성되어 있지 않으면 암호화 없이 연결이 설정됩니다. SSL 암호화를 사용하도록 SQL Server 인스턴스가 구성되어 있으면 Microsoft SQL Server JDBC 드라이버는 올바르게 구성된 JVM에서 실행되는 경우 자동으로 SSL 암호화를 사용합니다. 그렇지 않은 경우 연결이 종료되고 드라이버에서 오류가 발생합니다.
